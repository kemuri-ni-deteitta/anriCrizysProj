# Тренажёр антикризисного управления

## О программе

Desktop-приложение для студентов, симулирующее реальные кризисные ситуации в ИТ-компании. Пользователь читает описание проблемы, принимает решения на каждом шаге, объясняет свой выбор в текстовом поле, а после прохождения получает детальный разбор с указанием сильных сторон и зон для улучшения. По итогам можно сформировать отчёт в PDF или Word.

Цель — не оценка баллами, а первичная диагностика мышления студента: что он делает правильно, где есть слепые зоны, о чём стоит поговорить на занятии.

Приложение работает **полностью офлайн**, без подключения к интернету.

## Функциональность

| Экран | Описание |
|---|---|
| Регистрация | Ввод ФИО и группы при первом запуске; повторная авторизация не требуется |
| Главная (Dashboard) | Метрики активности: пройдено тестов, средний процент, последнее прохождение |
| Тесты | Выбор тематики → выбор теста → пошаговое прохождение с вариантами ответов |
| Результаты | Разбор по каждому вопросу, правильные ответы, комментарии, итоговый анализ |
| Мои результаты | История всех прохождений с фильтрацией по тематике |
| Календарь | Визуализация активности — дни с прохождениями выделены цветом |
| Отчёты | Генерация и скачивание отчёта по любому прохождению в форматах PDF или DOCX |
| Редактирование профиля | Изменение ФИО, группы, фотографии |
| Админ-панель | Создание, редактирование и удаление тестов и вопросов; статистика по студентам |

### Хранение данных

Никакой базы данных. Все данные — **плоские JSON-файлы** в папке `data/` рядом с приложением:

```
data/
├── user.json          — профиль текущего пользователя
├── avatars/           — фотографии профиля
├── sessions/          — история прохождений (по одному файлу на сессию)
└── tests/
    ├── topics.json    — список тематик
    ├── cybersecurity/ — тесты по кибербезопасности
    ├── hr_crisis/     — тесты по кадровым кризисам
    ├── financial/     — тесты по финансовым кризисам
    ├── operational/   — тесты по операционным сбоям
    ├── strategic/     — тесты по стратегическим просчётам
    ├── project/       — тесты по проектным кризисам
    └── regulatory/    — тесты по регуляторным кризисам
```

Запись в файлы **атомарная**: данные сначала записываются во временный файл, затем переименовываются — это защищает от повреждения при аварийном завершении.

### Коммуникация

Frontend (Electron/React) общается с Backend (FastAPI) через **REST API** на `localhost:8000`. Весь трафик остаётся на локальной машине.

## Сборка и запуск

### Разработка

```bash
# Backend
CRISIS_DATA_DIR=./data backend/.venv/bin/uvicorn main:app \
  --app-dir backend --host 127.0.0.1 --port 8000

# Frontend
npm run dev --prefix frontend
```

### Сборка в .exe (Windows)

```bash
npm run package:win --prefix frontend
```

## Роли пользователей

**Студент** — вводит ФИО и группу при первом запуске, после чего приложение его запоминает. Проходит тесты, просматривает результаты, скачивает отчёты.

**Администратор (преподаватель)** — вход по логину `admin`. Управляет содержимым тестов через Админ-панель, видит статистику по всем студентам на данной машине.

### 1. Главная (Dashboard)

Стартовый экран после входа. Даёт быстрый обзор активности студента.

- **Три карточки-метрики** вверху:
  - «Пройдено тестов» — количество завершённых тестов из общего числа
  - «Правильных ответов» — средний процент по всем прохождениям
  - «Последний тест» — название и дата последнего прохождения
- **Список «Последние прохождения»**: название теста, тематика (тег), дата, цветной индикатор результата (зелёный — хорошо, жёлтый — есть что улучшить)

---

### 2. Тесты

Двухуровневый выбор теста для прохождения.

**Уровень 1 — Выбор тематики.** Сетка карточек с иконкой, названием и количеством тестов внутри. Доступные тематики: Кибербезопасность, Кадровый кризис, Финансовый кризис, Операционный сбой, Стратегический просчёт, Проектный кризис, Регуляторный кризис.

**Уровень 2 — Список тестов тематики.** После клика на тематику отображается список тестов, каждый с названием, количеством вопросов, статусом («Не пройден» / «Пройден» / «Пройден повторно») и кнопкой «Начать».

**Экран прохождения теста.** После нажатия «Начать» открывается пошаговый режим:
- Вверху: название теста и прогресс («Вопрос 3 из 10») с прогресс-баром
- Текст ситуации / вопроса
- Варианты ответов (радио-кнопки для одиночного, чекбоксы для множественного выбора); у каждого варианта — текст, затраты по времени и стоимости, кнопка подсказки ℹ️
- Текстовое поле «Объясните свой выбор» (необязательно, попадает в отчёт)
- Кнопки «Назад» / «Далее»; после последнего вопроса — «Завершить тест»
- Студент может вернуться назад и изменить ответ, таймера нет

**Экран результатов теста.** Показывается сразу после завершения:
- Для каждого вопроса — карточка с вариантами ответов: правильный выделен зелёным, неверный выбор студента — другим цветом; ниже — пояснение к правильному ответу и использованные ресурсы (время + деньги)
- Итоговый текстовый анализ: «Что сделано верно» и «Что стоит улучшить»
- Кнопки: «Пройти снова» / «Перейти к отчётам» / «На главную»

---

### 3. Мои результаты

История всех прохождений текущего студента.

- Фильтр по тематике (табы или выпадающий список)
- Каждая строка: аватар-круг с процентом правильных ответов (цвет зависит от результата), название теста, тематика, дата прохождения, счёт («8 из 10»)
- Кнопка «Посмотреть разбор» открывает экран результатов для выбранного прохождения

---

### 4. Календарь

Визуализация активности студента по дням.

- Сетка календаря с помесячным отображением
- Дни, в которые были прохождения, выделены цветом
- Клик на день — всплывающий список тестов, пройденных в этот день
- Легенда: «тест пройден» / «сегодня»
- Навигация по месяцам (вперёд / назад)

---

### 5. Отчёты

Генерация документа по итогам любого прохождения.

- Выпадающий список для выбора теста из истории прохождений
- Предпросмотр содержимого отчёта: ФИО и группа студента, дата, название теста и тематика, все вопросы с ответами студента и правильными ответами, комментарии студента, итоговый анализ
- Выбор формата: **PDF** или **Word (DOCX)**
- Кнопка «Создать отчёт» — открывает диалог сохранения файла на диск

---

### 6. Админ-панель *(только для admin)*

Управление содержимым тестов и просмотр статистики. Доступна только администратору.

**Вкладка «Управление тестами»:**
- Список всех тестов с фильтром по тематике
- Кнопка «Добавить тест»
- Для каждого теста: название, тематика, количество вопросов, кнопки «Редактировать» и «Удалить»

**Вкладка «Редактор теста»:**
- Поля: название теста, тематика, описание кризисной ситуации
- Список вопросов с возможностью добавления и удаления
- Для каждого вопроса: текст, тип ответа (одиночный / множественный), до 4 вариантов ответа (текст, затраты по времени в часах, стоимость в рублях, текст подсказки, флаг «правильный»), пояснение к правильному ответу, итоговый анализ

**Вкладка «Статистика»:**
- Таблица: студент / тест / дата прохождения / процент правильных ответов

Кликабельный блок с именем и фотографией. При клике открывается модальное окно редактирования профиля:

- Аватарка и кнопка «Загрузить фото» (JPG/PNG)
- Поля: Фамилия, Имя, Группа
- Кнопки «Отмена» / «Сохранить»
