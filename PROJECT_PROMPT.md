# Тренажёр антикризисного управления — Промпт для Claude Code

---

## О проекте

Разрабатывается desktop-приложение-тренажёр для студентов, которое симулирует реальные кризисные ситуации в ИТ-компании. Пользователь читает описание проблемы, принимает решения по каждому шагу, объясняет свой выбор, а после получает детальный разбор и может сформировать отчёт. Приложение используется как учебный инструмент на курсах по стратегии, управлению рисками и антикризисному менеджменту.

Цель — не оценка баллами, а первичная диагностика мышления студента: что он делает правильно, где есть слепые зоны, о чём стоит поговорить на занятии.

---

## Технический стек

- **Frontend**: React Native (с возможностью сборки в `.exe` для Windows через Electron или аналог)
- **Backend**: Python (локальный сервис, офлайн, без интернета)
- **Хранение данных**: Flat file system — все данные хранятся в JSON-файлах на диске, без БД
- **Генерация отчётов**: Python-библиотеки для PDF (ReportLab или WeasyPrint) и Word (python-docx)
- **Коммуникация frontend ↔ backend**: REST API (localhost)
- **Хранение сессии**: локально в JSON-файлах, данные пользователя сохраняются между запусками

## Роли пользователей

### Студент (основная роль)
- Вводит Фамилию, Имя, Группу при первом запуске
- После этого приложение его запоминает — при повторном запуске авторизация не нужна
- Проходит тесты, смотрит результаты, скачивает отчёты

### Администратор (преподаватель)
- Логин: `admin`, пароль: `admin`
- Видит дополнительный пункт «Админ-панель» в навбаре
- Может добавлять, редактировать и удалять тесты, вопросы, варианты ответов
- Видит статистику по всем студентам на данной машине

## Экраны и их функционал

---

### Экран 0 — Первый запуск (регистрация)

Показывается **только один раз** — при первом запуске приложения.

**Содержимое:**
- Заголовок: название приложения / курса
- Три поля ввода: Фамилия, Имя, Группа
- Кнопка «Приступить»

**Логика:**
- После нажатия данные сохраняются локально
- При следующем запуске этот экран не показывается — пользователь сразу попадает на Главную
- Если пользователь — admin (особый логин), показывается отдельный экран входа с паролем

---

### Экран 1 — Главная (Dashboard)

**Назначение:** быстрый обзор активности студента.

**Содержимое:**
- Три карточки-метрики вверху:
  - «Пройдено тестов» (число / из общего количества)
  - «Правильных ответов» (средний % по всем тестам)
  - «Последний тест» (название + дата)
- Список «Последние прохождения»:
  - Название теста
  - Тематика (тег)
  - Дата прохождения
  - Цветной индикатор (зелёный = хороший результат, жёлтый = есть что улучшить)

---

### Экран 2 — Тесты

**Назначение:** выбор тематики и конкретного теста для прохождения.

**Структура (два уровня):**

#### Уровень 1 — Выбор тематики
Сетка карточек. Каждая карточка:
- Иконка / цветной маркер тематики
- Название тематики
- Количество тестов внутри

**Доступные тематики:**
1. Кибербезопасность
2. Кадровый кризис
3. Финансовый кризис
4. Операционный сбой
5. Стратегический просчёт
6. Проектный кризис
7. Регуляторный кризис

#### Уровень 2 — Выбор теста внутри тематики
После клика на тематику — список тестов. Каждый тест:
- Название (конкретная кризисная ситуация, н-р «Подозрительный трафик на сервере»)
- Количество вопросов
- Статус: «Не пройден» / «Пройден» / «Пройден повторно»
- Кнопка «Начать»

---

### Экран 3 — Прохождение теста (вопрос за вопросом)

**Структура страницы:**
- Вверху: название теста + прогресс (Вопрос 3 из 10)
- Прогресс-бар
- Описание ситуации / вопрос (крупный текст)
- Варианты ответов (радио-кнопки для одиночного / чекбоксы для множественного выбора)
  - У каждого варианта: текст + время ресурса + стоимость (справочно, не влияет на оценку) + кнопка ℹ️ (подсказка)
- Справа: текстовое поле «Объясните свой выбор» (необязательное, сохраняется в отчёт)
- Кнопки навигации: «Назад» / «Далее»
- Таймера нет

**Логика:**
- Тип ответа (одиночный / множественный) задаётся на уровне вопроса
- Студент может вернуться назад и изменить ответ
- После последнего вопроса — кнопка «Завершить тест»

---

### Экран 4 — Результаты теста

Показывается сразу после завершения.

**Содержимое:**
- Заголовок: название кризисной ситуации (красным или акцентным цветом)
- Для каждого вопроса — карточка:
  - Текст вопроса
  - Все варианты ответов
  - Выделение: правильный ответ (зелёный) / ответ студента (если неверный — другой цвет)
  - Пояснение к правильному ответу (текст из базы)
  - Использованные ресурсы (время + деньги)
  - Комментарий студента (если вводил)
- Итоговый текстовый анализ внизу:
  - Что сделано верно
  - Что стоит улучшить
- Кнопки: «Пройти снова» / «Перейти к отчётам» / «На главную»

---

### Экран 5 — Мои результаты

**Назначение:** история всех прохождений.

**Содержимое:**
- Фильтр по тематике (табы или дропдаун)
- Список прохождений — каждая строка:
  - Аватар-круг с % правильных ответов (цвет зависит от результата)
  - Название теста
  - Тематика
  - Дата прохождения
  - Количество верных ответов (н-р «8 из 10»)
  - Кнопка «Посмотреть разбор» → открывает Экран 4

---

### Экран 6 — Календарь

**Назначение:** визуализация активности — когда и что проходил студент.

**Содержимое:**
- Сетка календаря (помесячно)
- Дни с прохождениями — выделены цветом
- Клик на день → всплывающий список тестов, пройденных в этот день
- Легенда: «тест пройден» / «сегодня»
- Навигация по месяцам (вперёд / назад)

---

### Экран 7 — Отчёты

**Назначение:** генерация и скачивание отчёта по конкретному прохождению.

**Содержимое:**
- Дропдаун: выбор теста из истории прохождений
- Предпросмотр того, что войдёт в отчёт:
  - ФИО студента, группа, дата
  - Название теста и тематика
  - Все вопросы, ответы студента, правильные ответы
  - Комментарии студента
  - Итоговый анализ
- Выбор формата: PDF / Word (DOCX)
- Кнопка «Создать отчёт» → файл сохраняется на диск / открывается диалог сохранения

---

### Экран 8 — Редактирование профиля (модальное окно)

Открывается при клике на блок профиля внизу сайдбара.

**Содержимое:**
- Аватарка + кнопка «Загрузить фото» (JPG/PNG)
- Поля: Фамилия, Имя, Группа
- Кнопки: «Отмена» / «Сохранить»

---

### Экран 9 — Админ-панель (только для admin)

**Назначение:** управление контентом тестов.

**Вкладки:**

#### 9.1 Управление тестами
- Список всех тестов с фильтром по тематике
- Кнопка «Добавить тест»
- Для каждого теста: название, тематика, кол-во вопросов, кнопки «Редактировать» / «Удалить»

#### 9.2 Редактор теста
- Название теста
- Выбор тематики
- Описание кризисной ситуации
- Список вопросов:
  - Добавить / удалить вопрос
  - Для каждого вопроса:
    - Текст вопроса
    - Тип ответа (одиночный / множественный)
    - До 4 вариантов ответа, для каждого:
      - Текст
      - Время (ч)
      - Стоимость (₽)
      - Текст подсказки (ℹ️)
      - Флаг «правильный»
    - Пояснение к правильному ответу
    - Итоговый анализ (текст «что верно / что улучшить»)

#### 9.3 Статистика (опционально)
- Таблица: студент / тест / дата / % правильных

## Структура файловой системы (Flat File Storage)

Никакой базы данных. Все данные — JSON-файлы в папке `data/` рядом с приложением.

```
data/
├── user.json                        ← профиль текущего пользователя
├── sessions/
│   ├── session_001.json             ← одно прохождение теста
│   ├── session_002.json
│   └── ...
└── tests/
    ├── topics.json                  ← список тематик
    ├── cybersecurity/
    │   ├── test_suspicious_traffic.json
    │   ├── test_data_leak.json
    │   └── ...
    ├── hr_crisis/
    │   ├── test_key_dev_quit.json
    │   └── ...
    ├── financial/
    ├── operational/
    ├── strategic/
    ├── project/
    └── regulatory/
```

## Нефункциональные требования

- Приложение работает **полностью офлайн**, без интернета
- Все данные хранятся в **JSON-файлах** в папке `data/` рядом с приложением — никакой БД
- Backend — Python-сервис, читает и пишет JSON-файлы, запускается локально на `localhost`
- При первом запуске backend проверяет наличие папки `data/` и создаёт её со стартовым набором тестов если папки нет
- Запись в файлы атомарная: сначала пишем во временный файл, потом переименовываем — чтобы не было повреждений при сбое
- Приложение собирается в `.exe` для Windows
- Интерфейс — только на русском языке
- Минимальное разрешение: 1280×720

```
┌─────────────────────────────────────────────┐
│  SIDEBAR (220px)    │  MAIN CONTENT AREA     │
│                     │                        │
│  [ Лого / Название ]│  [ Topbar: заголовок ] │
│                     │  ───────────────────── │
│  [ Пункт меню 1 ]   │                        │
│  [ Пункт меню 2 ]   │  [ Контент страницы ]  │

│  [ Пункт меню 4 ]   │                        │
│  [ Пункт меню 5 ]   │                        │
│  ─────────────────  │                        │

│                     │                        │
```

### Компоненты UI
- **Карточки** — скруглённые, с тонкой рамкой, белый фон
- **Метрики** — карточки с крупным числом и подписью
- **Навигация** — активный пункт выделяется фоном и рамкой
- **Кнопки** — минималистичные, с рамкой; основное действие — акцентная заливка
- **Поля ввода** — тонкая рамка, фокус подсвечивается
- **Теги/бейджи** — пилюли с цветом тематики
- **Прогресс-бар** — тонкая линия под заголовком вопроса
- **Модальные окна** — поверх затемнённого фона, центрированы
- **Индикаторы результата** — цветные круги/точки (хорошо / средне / плохо)

### Принципы
- Минимализм: много воздуха, без декоративного шума
- Иерархия через размер текста и насыщенность, не через цвет
- Один акцентный цвет на всё приложение
- Все интерактивные элементы имеют состояния: default / hover / active / disabled

## Что НЕ входит в scope v1.0

- Сетевой мультиплеер / облако
- Авторизация через внешние сервисы
- Мобильная версия
- Уведомления / push
- Импорт тестов из Excel/CSV (можно добавить в v2)
- Рейтинг между студентами

## Порядок разработки (рекомендуемый)

1. Инициализация проекта (React Native + Electron + Python backend)
2. Flat file система: структура папок `data/`, seed-данные (1 тест с вопросами в JSON)
3. Python-сервис: утилиты чтения/записи JSON + REST API эндпоинты
4. Экран регистрации + сохранение `user.json`
5. Sidebar + роутинг между экранами
6. Экран «Тесты» (тематики → список → начало)
7. Экран прохождения теста (вопрос за вопросом + запись `session_*.json`)
8. Экран результатов (чтение session + теста)
9. Главная (дашборд — агрегация по session-файлам)
10. Мои результаты + Календарь
11. Отчёты (PDF + DOCX)
12. Редактирование профиля (обновление `user.json` + аватарка)
13. Админ-панель (CRUD над JSON-файлами тестов)
14. Полировка дизайна + сборка в `.exe`
